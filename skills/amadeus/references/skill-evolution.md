# Skill Evolution

Improve transferable behavior with the smallest justified control-surface change. Use an independent SkillEvolutionExpert when reusable changes affect topology, autonomy, safety, observation, review authority, interaction, or transfer. The expert diagnoses but does not edit, self-review, see hidden keys before precommitment, or authorize installation.

Classify the locus as `policy_gap`, `policy_ambiguity`, `activation_failure`, `execution_failure`, `verification_gap`, `capability_or_environment_gap`, or `interaction_failure`. Permanent policy requires a demonstrated gap, ambiguity, or reproducible interaction failure.

## Choose The Smallest Evidence Mode

Use `compact` for a narrow, additive, incident-backed interaction clarification that does not relax or redesign topology, autonomy, safety, privacy, authority, capability floors, protected effects, or repository protections. Freeze the observed baseline failure, corrected incident, one surface-distant transfer, one non-example or protected-policy interaction, a short rubric and hard failures, one independent scorer, and at most one retry after the candidate or evidence changes. The baseline may be the attributable raw incident trace; do not spend tokens recreating a failure already observed.

Use `full` when the change is broad, subtractive, changes protected behavior or agent authority, lacks a trustworthy incident baseline, claims general superiority, or fails/collides in the compact cycle. Do not run the full suite merely because it exists. In either mode, every check must name the behavior or protected invariant it can falsify; delete checks with no decision value.

## EvolutionEvidencePlan

Create this plan before the first material reusable-policy edit and reserve its candidate and scorer nodes in the runtime DAG. Deterministic implementation, formatting, and package-sync changes with unchanged policy use objective checks instead:

```text
mode: compact | full
behavioral_claim_and_failure_locus
unchanged_skill_snapshot_and_candidate_snapshot
acceptance_criteria_and_hard_failures
corrected_incident
transfer_cases: one for compact | at least two surface-distant for full
non_example_or_protected_interaction: one combined case allowed for compact
untouched_confirmation: required for full | required for compact only when no attributable incident baseline
baseline_and_candidate_execution_contracts
independent_blind_scorer_and_rubric
affected_case_invalidation_and_rerun_rule
promotion_or_rejection_disposition
```

Freeze tasks and scoring before candidate exposure. When compact mode uses an attributable incident trace as baseline, bind that exact trace and run the candidate on the frozen corrected incident and transfer cases; otherwise baseline and candidate respondents receive the same raw tasks but not answer keys or each other's output. The scorer receives outputs and the frozen rubric without candidate authorship. Compare behavior, false alarms, latency/topology cost, and protected-invariant regressions—not keyword presence. A policy is still experimental when execution is infeasible; label it so rather than treating prose review as validation.

Review the controlling policy/diff, impact map, invariants, and test topology before verification. Run structural and focused mechanism/collision checks first; launch broad activation and transfer suites only when the full mode or a compact-path failure requires them. After failure, rerun affected focused cases only when candidate, input, environment, hypothesis, or evidence changed; final disposition and the selected mode's required confirmation remain mandatory. Never rerun an unchanged full suite merely because capacity is available.

Apply entailment, deletion, substitution, and collision tests. Maintain the protected-invariant register. A compact candidate must pass its corrected incident, transfer, non-example/protected interaction, unchanged-skill comparison, and independent score; a full candidate also requires the additional transfers and untouched confirmation above. Map every atomic post-run edit to each case's producer, effective input, evaluator, topology, freshness, and protected-gate closure using [evidence-provenance-and-reuse.md](evidence-provenance-and-reuse.md). Rerun affected producer/gate closures, rescore evaluator-only changes, and cite checked equivalence for retained evidence. Unknown reachability blocks only mapped claims pending the cheapest dependency probe. A rubric/key change after result exposure cannot satisfy untouched confirmation. Keep prompts and keys outside the installed package.
